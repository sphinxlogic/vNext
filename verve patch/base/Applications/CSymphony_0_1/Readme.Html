Csy is a utility for generating code in multiple .Net languges by building the Code Graph in any one .Net language.

It uses the CodemDOM wrapper CSymphony.Lib. Unlike the other CodeDOM libraries the code graph can be built in a more natural programming style. The library maintains the stack for the common statement containers, eg. Classes, Methods, Iterations, Exceptions, etc.

Please see the sample folders (vbSamples, csSamples) for around 50 samples in VB.NET and C# for various code / data type constructs. Read Quick Start for a fast-paced intro the the CSy utility.

Also see the Samples with the input code in C# & VB.net and the resulting output generated in C#, VB.net, J#, JScript, C++.net.

The folders vbSamples, csSamples include the batch file to test the samples. The batch files use the following command line :

csy -i<FileName.ext> -p:CS

or

csy -i<FileName.ext> -p:VB

Just click on the batch files and the code is generated in the the SourceOut folder. To add/remove languages or to change the destination folder for the generated code edit the csy.xml file in these folders. (see the sample xml config file below for more clues).

Usage :

csy -c<csy.xml> -i<inputFile> -p<providerID>

all the 3 parameters viz. -c, -i, -p are optional. See the sample usage below for more details.

csy uses the input file to generate code in multiple .net languages as specified in the <codeGen> tag. The resultant output file(s) is created in the folder specified by the path attribute specified in the <codegen> tag.

The <codeProviders> tag lists the .Net languages to be used as an input language or output languages.

The input file is compiled using the Code Provider as specified in the -p parameter or the prvId tag of the <in> tag.

The following is the sample Xml file.

<?xml version="1.0" standalone="yes"?>
<csy>
<in file="CodeConditionStatement.csy" prvId="CS"/>
<codeGen>
<out path="SourceOut" prvId="CS"/>
<out path="SourceOut" prvId="VB"/>
<out path="SourceOut" prvId="CPP"/>
<out path="SourceOut" prvId="JSCRIPT"/>
<out path="SourceOut" prvId="JSHARP"/>
</codeGen>
<codeProviders>
<codeProvider prvId="CS" fileExt="cs" info= "C#" assembly="System, Version=1.1.4322.0, Culture=neutral, PublicKeyToken=b77a5c561934e089" providerType = "Microsoft.CSharp.CSharpCodeProvider" />

<codeProvider prvId="VB" fileExt="vb" info= "Visual Basic .NET" assembly="System, Version=1.0.3300.0, Culture=neutral, PublicKeyToken=b77a5c561934e089" providerType = "Microsoft.VisualBasic.VBCodeProvider" />

<codeProvider prvId="JSHARP" fileExt="jsl" info= "JSharp" assembly="VJSharpCodeProvider, Version=7.0.5000.0, Culture=neutral, PublicKeyToken=b03f5f7f11d50a3a" providerType = "Microsoft.VJSharp.VJSharpCodeProvider" />

<codeProvider prvId="CPP" fileExt="cpp" info= "C++.net" assembly="MCppCodeDomProvider, Version=7.0.5000.0, Culture=neutral, PublicKeyToken=b03f5f7f11d50a3a" providerType="Microsoft.MCpp.MCppCodeProvider" />

<codeProvider prvId="JSCRIPT" fileExt="js" info= "Jscript.Net" assembly="Microsoft.JScript, , Version=7.0.5000.0, Culture=neutral, PublicKeyToken=b03f5f7f11d50a3a" providerType = "Microsoft.JScript.JScriptCodeProvider" />

</codeProviders>

</csy>

Notes :

When the -c<configFile> option is absent csy.xml is assumed to be the config file.

When the -i<inFile> option is absent , the file attribute of the <in> tag is used as the input file.

For eg. <in file="myInput.vb">

When th -p<ProviderID> option is absent the prvId attribute of the <in> tag is used as the Code Provider for the input file.

For eg. <in file="myInput.graph" prvId="VB">

All the providers used as an input or outpur are to be specified withing the codeprovider tags.

sample usage :

a) csy

takes the config details from the csy.xml file, uses the <in file> and the <in prvId> of the csy.xml file as the input file and the provider id, for eg. <in file="MyGraph.cs" prvId="CS" />

b) csy -c:MyConfigFile.xml

Same as (a) except the details come from MyConfigFile.xml instead of Csy.xml.

c) csy -i:MyGraph.cs -p:CS

Same as (a) except the input file and the provider Id is specified. Useful when the same config file is to be used for multiple input file. Prevents you from creating multiple config files.

d) csy -c:MyConfigFile -i:MyGraph.vb -p:VB
